En una instalación de un solo árbol, cualquier persona puede registrar una nueva cuenta en una instancia de Gramps Web, pero necesitará ser habilitada manualmente por un propietario del sitio antes de poder iniciar sesión.

En una instalación de múltiples árboles, los usuarios además requieren el ID del árbol para registrarse; en la práctica, esto significa que pueden registrarse utilizando un enlace de registro compartido por el propietario del árbol.

## 1. Encontrar el formulario de registro

En el formulario de inicio de sesión en la página de inicio de Gramps Web, generalmente hay un enlace "Registrar nueva cuenta".

Si falta este enlace, probablemente se deba a que es una instalación de múltiples árboles de Gramps Web. En ese caso, contacta al propietario del árbol para obtener un enlace de registro.

El propietario del árbol puede encontrar y copiar el enlace de registro en Configuración/Administración.

## 2. Enviar el formulario de registro

El formulario de registro tiene cuatro campos obligatorios: nombre de usuario, contraseña, dirección de correo electrónico y nombre completo. Cuando se hayan completado los cuatro campos, haz clic en "enviar" para iniciar el proceso de registro.

## 3. Confirmar la dirección de correo electrónico

Después de que se haya enviado el formulario, se enviará un correo electrónico automatizado a la dirección de correo electrónico proporcionada que contiene un enlace de confirmación de correo electrónico. Al hacer clic en él, se abrirá una página de confirmación.

Si el correo electrónico de confirmación no llega (y has revisado tu carpeta de Spam), podría ser que el correo electrónico no ha sido configurado correctamente por el administrador del servidor de Gramps Web.

## 4. Habilitar cuenta

Después de que se haya confirmado la dirección de correo electrónico, el propietario del árbol recibirá una notificación por correo electrónico y podrá habilitar la nueva cuenta de usuario yendo a la sección "Administración de usuarios" en la página de configuración (accesible a través del ícono de usuario en la barra superior de la aplicación) y cambiando el rol del usuario de "deshabilitado" a cualquiera de los [otros roles](../install_setup/users.md).

Ten en cuenta que el nuevo usuario *no* recibirá una notificación automatizada de que su cuenta ha sido habilitada, por lo que es recomendable que el administrador les notifique personalmente.
