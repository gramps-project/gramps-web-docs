# Створіть обліковий запис для власника дерева

Перш ніж ви зможете почати використовувати Gramps Web, вам потрібно створити обліковий запис для власника дерева. Якщо для даного дерева не існує облікового запису користувача, буде показана форма для створення облікового запису. Форма залежить від налаштування сервера, яке може бути для одного дерева або для кількох дерев.

## Налаштування для одного дерева: створити обліковий запис адміністратора

На сервері з налаштуванням для одного дерева, коли обліковий запис користувача ще не існує, відкриття Gramps Web показує форму для створення облікового запису адміністратора. Користувач-адміністратор буде як власником (єдиного) дерева, так і адміністратором установки. Форма також дозволяє налаштувати конфігурацію електронної пошти, необхідну для сповіщень по електронній пошті (наприклад, скидання пароля користувача). Якщо конфігурація електронної пошти вже була додана через файл конфігурації або змінні середовища на сервері, цю частину форми можна залишити пустою.

## Налаштування для кількох дерев: створити обліковий запис адміністратора

У налаштуванні для кількох дерев буде показана та ж форма для створення облікового запису адміністратора, якщо користувачі не існують *в жодному дереві*, тобто коли сервер тільки що був створений.

## Налаштування для кількох дерев: створити обліковий запис власника дерева

У налаштуванні для кількох дерев кожен користувач прив'язаний до єдиного дерева. Навіть якщо користувачі вже існують в інших деревах, обліковий запис власника дерева може бути створений у веб-інтерфейсі, якщо власника *для цього дерева* ще не існує.

Однак форма створення власника не буде відображатися автоматично на домашній сторінці Gramps Web, яка є такою ж для всіх дерев. Натомість до неї можна дістатися за адресою `https://my-gramps-instance/firstrun/my-tree-id`, де `https://my-gramps-instance` є базовою адресою вашої установки Gramps Web, а `my-tree-id` є ID вашого дерева.

Можливий робочий процес для адміністратора сайту для створення нового дерева:

- Створити дерево через REST API, отримавши ID нового дерева
- Поділитися посиланням на форму створення власника з відповідним ID дерева з потенційним власником дерева

Форма створення власника дерева аналогічна формі створення адміністратора, описаній вище, за винятком того, що вона не дозволяє змінювати конфігурацію електронної пошти (що дозволено лише для адміністраторів).
