# Настройки администрирования

Страница **Настройки > Администрирование** доступна через иконку пользователя в верхней панели приложения. Она доступна только пользователям с ролью Владельца или Администратора и предоставляет инструменты для управления базой данных семейного дерева.

Страница организована в сворачиваемые секции. Нажмите на заголовок секции, чтобы развернуть её.

## Данные

Охватывает квоты использования, импорт данных и управление медиафайлами.

### Квоты использования

В верхней части секции отображается текущее использование относительно любых настроенных лимитов:

- **Люди** – количество объектов людей в дереве по сравнению с настроенным максимумом (∞ если безлимитно)
- **Хранение медиа** – общий размер загруженных медиафайлов по сравнению с настроенной квотой хранения (∞ если безлимитно)

Квоты устанавливаются администратором сервера; смотрите [Конфигурация сервера](../install_setup/configuration.md) для получения подробной информации.

### Импорт данных

Секция импорта позволяет загружать файл семейного дерева или медиаархив. Смотрите [Импорт данных](import.md) для получения полных инструкций.

### Статус медиафайлов

Эта секция показывает:

- Общее количество медиаобъектов в дереве и наличие объектов без контрольной суммы
- Количество медиаобъектов, связанные файлы которых отсутствуют на сервере

Зеленая галочка указывает на то, что все в порядке. Если обнаружены проблемы, отображаются ссылки на затронутые объекты. Отсутствие контрольных сумм обычно происходит, когда данные были импортированы из формата, такого как GEDCOM, который включает ссылки на медиа, но не сами файлы. Отсутствующие файлы можно загрузить через функцию импорта медиаархива.

### Импорт медиаархива

Позволяет загружать ZIP-файл медиафайлов для заполнения отсутствующих файлов. Смотрите [Импорт данных](import.md) для получения полных инструкций.

## Индекс поиска

### Управление индексом поиска

Gramps Web поддерживает полнотекстовый индекс поиска, который обычно обновляется автоматически при изменении данных. Индикатор статуса показывает, сколько объектов в настоящее время индексировано по сравнению с общим количеством объектов.

Нажмите **Обновить индекс поиска**, чтобы инициировать полное восстановление. Индикатор прогресса отображается, пока задача выполняется в фоновом режиме. Это обычно требуется только после обновления сервера.

### Семантический индекс поиска

Если на сервере включен [семантический (с поддержкой ИИ) поиск](../install_setup/configuration.md), появляется дополнительная секция с двумя действиями:

- **Регенерировать семантический индекс поиска** – восстанавливает весь семантический индекс с нуля. Это требует значительных вычислительных ресурсов и может занять много времени.
- **Обновить семантический индекс поиска** – выполняет инкрементальное обновление, добавляя только объекты, которые еще не индексированы. Быстрее, чем полное восстановление.

## Настройки дерева

### Имя семейного дерева

!!! note
    Переименование дерева работает только в [мульти-дереве](../install_setup/multi-tree.md) или когда `TREE_ID` явно установлен в [конфигурации сервера](../install_setup/configuration.md). В стандартной установке с одним деревом без установленного `TREE_ID` это вызовет ошибку.

Это позволяет изменить имя базовой базы данных семейного дерева Gramps. Введите новое имя и нажмите **Переименовать**, чтобы применить.

!!! tip
    Если вы хотите изменить только имя, отображаемое в панели приложения, не переименовывая базу данных, используйте настройку [Название приложения](#app-title).

### Информация о исследователе

Установите имя, адрес и контактные данные основного исследователя. Эта информация встраивается в экспорты (например, файлы GEDCOM).

## Настройка

### Цвета темы

Установите пользовательский **основной цвет** и **акцентный цвет** для интерфейса Gramps Web. Эти цвета применяются ко всем пользователям этого дерева и вступают в силу немедленно после сохранения.

Используйте выбор цветов, чтобы выбрать цвета, затем нажмите **Сохранить**. Нажмите **Сбросить**, чтобы вернуть значения по умолчанию.

### Название приложения

Установите пользовательское название для приложения. Если установлено, это переопределяет имя семейного дерева в заголовке браузера и верхней панели приложения.

Введите название и нажмите **Сохранить**. Оставьте пустым, чтобы использовать значение по умолчанию (имя семейного дерева).

### Заметка на главной странице

Выберите объект **Заметка** Gramps для отображения на главной странице панели управления. Содержимое заметки отображается под основными колонками панели управления и видно всем пользователям, имеющим доступ к дереву.

Используйте селектор объектов, чтобы найти и выбрать заметку, затем сохраните. Нажмите **Удалить**, чтобы очистить текущую заметку на главной странице.

### Изображение на главной странице

Выберите объект **Медиа** Gramps для отображения в качестве изображения на главной странице панели управления. В сочетании с заметкой на главной странице изображение отображается рядом с текстом заметки. Без заметки отображается только изображение.

Используйте селектор объектов, чтобы найти и выбрать медиаобъект, затем сохраните. Нажмите **Удалить**, чтобы очистить текущее изображение на главной странице.

### Настройки экспорта/импорта

Настройки уровня дерева (название приложения, цвета темы, заметка/изображение на главной странице и т. д.) могут быть экспортированы в файл JSON для резервного копирования или копирования в другой экземпляр Gramps Web.

- Нажмите **Экспортировать настройки**, чтобы скачать текущие настройки в виде файла JSON.
- Нажмите **Импортировать настройки дерева**, чтобы загрузить ранее экспортированный файл JSON и применить настройки.

## Обработка семейного дерева

### Проверка и восстановление базы данных

Этот инструмент проверяет базу данных Gramps на внутренние несоответствия и исправляет те, которые может – аналогично инструменту [Проверка и восстановление базы данных](https://www.gramps-project.org/wiki/index.php/Gramps_5.2_Wiki_Manual_-_Tools#Check_and_Repair_Database) в Gramps Desktop.

Нажмите **Проверить и восстановить** и дождитесь завершения индикатора прогресса. Результат отображается под кнопкой:

- Если ошибок не найдено, отображается сообщение подтверждения.
- Если ошибки найдены, отображается сводка примененных исправлений.

Запустите этот инструмент, если вы столкнулись с неожиданными ошибками или поведением, которое может быть вызвано несоответствиями в базе данных, такими как отсутствующие отношения между объектами.

## Теги

### Управление тегами

Создавайте, переименовывайте, изменяйте цвет и удаляйте [теги](../user-guide/tags.md) для семейного дерева. Теги хранятся в базе данных Gramps, общие для всех пользователей и полностью совместимы с Gramps Desktop.

Нажмите **Новый тег**, чтобы создать тег. Используйте элементы управления рядом с существующим тегом, чтобы переименовать его (иконка карандаша), изменить его цвет (выбор цвета) или удалить его (иконка удаления).

!!! note
    Удаление тега удаляет его из всех объектов, к которым он был применен.

Смотрите [Теги](../user-guide/tags.md) для получения информации о том, как теги используются в Gramps Web, включая специальные теги `Blog` и `ToDo`.

## Опасная зона

!!! danger
    Действия в Опасной зоне **необратимы**. Сделайте резервную копию перед продолжением.

### Удалить все объекты

Удаляет объекты из семейного дерева. Нажатие на **Удалить** открывает диалог, в котором вы можете выбрать, что удалить:

- **Все объекты** – полностью очищает дерево
- **Конкретные типы объектов** – например, только события или только медиаобъекты

Вам будет предложено повторно аутентифицироваться (войти снова), чтобы подтвердить действие. Удаление выполняется как фоновая задача, и отображается индикатор прогресса.

!!! warning
    Удаление только подмножества типов объектов (вместо всех объектов сразу) может занять очень много времени для больших деревьев, так как сервер должен проверить и обновить все отношения между объектами.

!!! tip
    Используйте это, чтобы начать с нуля перед импортом нового дерева или для удаления конкретных типов объектов, которые были импортированы неправильно.
