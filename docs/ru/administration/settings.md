# Настройки администрирования

Страница **Настройки > Администрирование** доступна через значок пользователя в верхней панели приложения. Она доступна только пользователям с ролью Владельца или Администратора и предоставляет инструменты для управления базой данных семейного дерева.

## Квоты использования

В верхней части страницы отображается текущее использование относительно любых настроенных лимитов:

- **Люди** – количество объектов человек в дереве по сравнению с настроенным максимумом (∞ если неограничено)
- **Хранение медиа** – общий размер загруженных медиафайлов по сравнению с настроенной квотой хранения (∞ если неограничено)

Квоты устанавливаются администратором сервера; см. [Конфигурация сервера](../install_setup/configuration.md) для получения подробной информации.

## Импорт данных

Раздел импорта позволяет загружать файл семейного дерева или архив медиафайлов. См. [Импорт данных](import.md) для получения полных инструкций.

## Статус медиафайлов

В этом разделе отображается:

- Общее количество медиаобъектов в дереве и наличие объектов без контрольной суммы
- Количество медиаобъектов, связанные файлы которых отсутствуют на сервере

Зеленая галочка указывает на то, что все в порядке. Если обнаружены проблемы, отображаются ссылки на затронутые объекты. Отсутствующие контрольные суммы обычно возникают, когда данные были импортированы из формата, такого как GEDCOM, который включает ссылки на медиа, но не сами файлы. Отсутствующие файлы можно загрузить через функцию импорта медиаархива.

## Импорт медиаархива

Позволяет загрузить ZIP-файл медиафайлов для заполнения отсутствующих файлов. См. [Импорт данных](import.md) для получения полных инструкций.

## Управление индексом поиска

Gramps Web поддерживает полнотекстовый индекс поиска, который обычно обновляется автоматически при изменении данных. Индикатор состояния показывает, сколько объектов в настоящее время индексируется по сравнению с общим количеством объектов.

Нажмите **Обновить индекс поиска**, чтобы инициировать полное восстановление. Индикатор прогресса отображается во время выполнения задачи в фоновом режиме. Это обычно требуется только после обновления сервера.

### Индекс семантического поиска

Если на сервере включен [семантический (с использованием ИИ) поиск](../install_setup/configuration.md), появляется дополнительный раздел с двумя действиями:

- **Регенерировать индекс семантического поиска** – восстанавливает весь семантический индекс с нуля. Это требует значительных вычислительных ресурсов и может занять много времени.
- **Обновить индекс семантического поиска** – выполняет инкрементальное обновление, добавляя только объекты, которые еще не индексированы. Быстрее, чем полное восстановление.

## Имя семейного дерева

!!! note
    Переименование дерева работает только в [настройке с несколькими деревьями](../install_setup/multi-tree.md) или когда `TREE_ID` явно установлен в [конфигурации сервера](../install_setup/configuration.md). В стандартной установке с одним деревом без установленного `TREE_ID` это вызовет ошибку.

Это позволяет изменить имя базового семейного дерева Gramps. Введите новое имя и нажмите **Переименовать**, чтобы применить.

## Проверка и восстановление базы данных

Этот инструмент проверяет базу данных Gramps на внутренние несоответствия и исправляет те, которые он может – аналогично инструменту [Проверка и восстановление базы данных](https://www.gramps-project.org/wiki/index.php/Gramps_5.2_Wiki_Manual_-_Tools#Check_and_Repair_Database) в Gramps Desktop.

Нажмите **Проверить и восстановить** и дождитесь завершения индикатора прогресса. Результат отображается под кнопкой:

- Если ошибок не найдено, отображается сообщение с подтверждением.
- Если ошибки были найдены, отображается сводка примененных исправлений.

Запустите этот инструмент, если вы столкнулись с неожиданными ошибками или поведением, которое может быть вызвано несоответствиями в базе данных, такими как отсутствующие связи между объектами.

## Опасная зона

!!! danger
    Действия в Опасной зоне **необратимы**. Сделайте резервную копию перед продолжением.

### Удалить все объекты

Удаляет объекты из семейного дерева. Нажатие на **Удалить** открывает диалог, в котором вы можете выбрать, чтобы удалить:

- **Все объекты** – полностью очищает дерево
- **Определенные типы объектов** – например, только события или только медиаобъекты

Вам будет предложено повторно пройти аутентификацию (войти снова), чтобы подтвердить действие. Удаление выполняется как фоновая задача, и отображается индикатор прогресса.

!!! warning
    Удаление только подмножества типов объектов (вместо всех объектов сразу) может занять очень много времени для больших деревьев, так как сервер должен проверить и обновить все связи между объектами.

!!! tip
    Используйте это, чтобы начать с чистого листа перед импортом нового дерева или чтобы удалить определенные типы объектов, которые были импортированы неправильно.
