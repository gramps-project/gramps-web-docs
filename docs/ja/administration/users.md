# ユーザー管理

ユーザー管理インターフェースは、**設定 > ユーザー管理**（アプリバーの上部にあるユーザーアイコン）からアクセスできます。これは、オーナーまたは管理者の役割を持つユーザーのみが利用可能です。

## ユーザー役割

利用可能なユーザー役割とその権限の完全な説明については、[ユーザーシステム](../install_setup/users.md)を参照してください。

## ユーザーの表示とフィルタリング

ユーザー管理ページには、以下の列を持つすべての登録済みユーザーアカウントのテーブルが表示されます。

- **ユーザー名** — ログイン名
- **フルネーム** — 表示名
- **Eメール** — ユーザーのEメールアドレス
- **役割** — 割り当てられた役割（ゲスト、メンバー、寄稿者、編集者、オーナー、または管理者）
- **アカウントソース** — "パスワード"（ローカルアカウント）または外部IDプロバイダーの名前（例：OIDCを使用している場合）

テーブルの上部にある検索フィールドと役割のドロップダウンを使用してリストをフィルタリングします。フィルタークリアボタンをクリックしてすべてのフィルターをリセットします。

## ユーザーの編集

任意の行の編集（鉛筆）アイコンをクリックして編集ダイアログを開きます。ユーザーの以下の情報を変更できます：

- フルネーム
- Eメールアドレス
- 役割

これは**新しく自己登録したユーザーを有効にする**主な方法です：彼らの役割を*無効*から任意のアクティブな役割（例：メンバーまたは編集者）に変更します。

## ユーザーを手動で追加

テーブルの上にある**ユーザーを追加**（人を追加）アイコンをクリックして、自己登録を必要とせずに新しいユーザーアカウントを直接作成します。ダイアログにユーザー名、フルネーム、Eメールアドレス、パスワード、および役割を入力し、**保存**をクリックします。

## ユーザーの削除

任意の行の削除（ゴミ箱）アイコンをクリックし、ダイアログを確認します。この操作は元に戻せません。

## ユーザーアカウントのエクスポートとインポート

これらのボタンは、[別のGramps Webインスタンスに移行する際](export.md)に便利です。

- **ユーザー詳細をエクスポート**（ダウンロードアイコン） — すべてのユーザーアカウントを含むJSONファイルをダウンロードします（パスワードは暗号化された形式で保存されているため含まれません）。
- **ユーザーアカウントをインポート**（グループ追加アイコン） — 以前にエクスポートしたJSONファイルをアップロードして、ユーザーアカウントを一括作成します。すべてのインポートされたユーザーは、パスワードを転送できないため、「パスワードを忘れた」リンクを介して新しいパスワードを設定する必要があります。

## 登録リンク（マルチツリーセットアップのみ）

マルチツリーセットアップでは、新しいユーザーのための登録リンクがユーザー管理ページの上部に表示されます。このリンクをコピーして、アカウント登録に招待したい人と共有できます。

!!! note
    シングルツリーセットアップでは、ログインページに一般的な「登録」リンクがあります。ツリーごとの登録リンクは、マルチツリーインストールでのみ必要です。

## AIチャットの権限

サーバーでAIチャットが有効になっている場合、ページの上部にあるドロップダウンで、どのユーザー役割がチャット機能を使用できるかを制御できます：

- すべての人（ゲストを含む）
- メンバー以上
- 寄稿者以上
- 編集者以上
- オーナーおよび管理者のみ
- 誰も（すべてのユーザーのチャットを無効にする）
