# 通知

**通知**はベルアイコンを持つサイドバー項目です。エラーが発生したり、バックグラウンドタスクが実行中のとき、バッジが未読通知の数を表示します。それをクリックすると通知ログが開きます。

通知ログは二つの目的を果たします：

- あなたのセッション中に発生したエラーの記録です – 失敗したAPIリクエスト、バックグラウンドタスクのエラー、保存失敗、またはブラウザレベルのエラー。
- 長時間実行されるバックグラウンドタスクの進行状況を追跡します – インポートやエクスポート、レポートの生成、OCRテキスト認識、データベースのアップグレード、検索/セマンティックインデックスの再構築など – それらの状態（例：保留中、開始、進行中）を表示し、完了または失敗した際に通知します。

各エントリには短いメッセージ、ソース（ネットワーク、タスク、保存、またはブラウザ）、およびタイムスタンプが表示されます。

一部の通知には構造化された詳細が含まれています。そのようなエントリをクリックすると、エラーデータの内訳と**JSONをコピー**ボタンを含むダイアログが開きます。これはバグを報告する際に便利で、JSONにはサーバーからの正確なエラー情報が含まれています。

**すべてクリア**を使用して、すべての通知を消去します。

!!! note
    通知はメモリにのみ保存され、ページをリロードするとクリアされます。
