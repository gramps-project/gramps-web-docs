# 使用内置任务管理

Gramps Web 包含一个内置的家谱任务管理工具。它旨在使研究人员能够规划和优先处理任务，同时记录他们的任务。这就是为什么任务在 Gramps 数据库中被表示为来源。在完成任务后，相关内容可以作为记录研究过程的来源。

## 任务基础

任务具有以下属性：

- 状态。这可以是“开放”、“进行中”、“阻塞”或“完成”
- 优先级。这可以是“低”、“中”或“高”
- 标签。这些标签是基础来源的普通 Gramps 标签。（请注意，所有任务还附加有 `ToDo` 标签以识别它们为任务，但该标签在任务列表中是隐藏的，以避免杂乱。）
- 标题。在任务列表中显示
- 描述。一个富文本字段，可用于描述问题陈述，也可以记录任何进展
- 媒体。附加到任务的任何媒体文件

## 创建任务

由于任务是普通的 Gramps 对象，因此可以由能够编辑或创建其他对象（如人员或事件）的同一组用户进行编辑或创建。

要创建任务，请在任务列表页面上单击 + 按钮。至少输入一个标题。创建时状态将始终为“开放”。

## 编辑任务

要编辑任务的任何详细信息，请在任务列表中单击它。

任务详细信息页面没有像其他 Gramps 对象那样的单独“编辑模式”。标题、状态和优先级的更改会立即应用。对富文本描述的更改需要单击其下方的“保存”按钮。

## 批量更改任务属性

可以通过使用任务列表中的复选框进行选择和任务列表上方的适当按钮来批量更改任务的优先级和状态。

## Gramps Desktop 中的任务

通过 Gramps Web 添加任务时，来源和备注将附加 `ToDo` 标签，因此任务将显示在桌面 [待办事项备注 Gramplet](https://gramps-project.org/wiki/index.php/Addon:ToDoNotesGramplet) 和 [待办事项报告](https://gramps-project.org/wiki/index.php/Addon:ToDoReport) 中。

要在 Gramps Desktop 中添加或编辑任务，请遵循以下指南：

- 添加一个带有 `ToDo` 标签的来源，并将任务标题作为标题
- 可选地，向带有 `ToDo` 标签的来源添加一个备注，类型为“待办事项”，描述为文本
- 添加一个属性“状态”，并将其设置为“开放”、“进行中”、“阻塞”或“完成”
- 添加一个属性“优先级”，并将其设置为 9（低）、5（中）或 1（高）（这些反直觉的值取自 iCalendar 规范）
