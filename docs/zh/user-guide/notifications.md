# 通知

**通知** 是一个带有铃铛图标的侧边栏项目。当发生错误或后台任务正在运行时，徽章会显示未读通知的数量。点击它以打开通知日志。

通知日志有两个目的：

- 它记录了您会话期间发生的错误——失败的 API 请求、后台任务错误、保存失败或浏览器级错误。
- 它跟踪长时间运行的后台任务的进度——例如导入和导出、生成报告、OCR 文本识别、数据库升级和搜索/语义索引重建——显示它们的状态（例如，待处理、已开始、进行中）并在它们完成或失败时通知您。

每个条目显示一条简短消息、来源（网络、任务、保存或浏览器）和时间戳。

一些通知包含结构化细节。点击此类条目会打开一个对话框，显示错误数据的详细信息和一个 **复制 JSON** 按钮。这在报告错误时非常有用，因为 JSON 包含来自服务器的确切错误信息。

使用 **清除所有** 来关闭所有通知。

!!! note
    通知仅存储在内存中，并在您重新加载页面时被清除。
