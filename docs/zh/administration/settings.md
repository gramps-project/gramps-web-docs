# 管理设置

**设置 > 管理** 页面可以通过应用程序顶部的用户图标访问。此页面仅对拥有所有者或管理员角色的用户可用，提供管理家谱数据库的工具。

## 使用配额

页面顶部显示当前使用情况与任何配置限制的关系：

- **人员** — 树中人员对象的数量与配置的最大值（如果无限制则为∞）
- **媒体存储** — 上传的媒体文件的总大小与配置的存储配额（如果无限制则为∞）

配额由服务器管理员设置；有关详细信息，请参见 [服务器配置](../install_setup/configuration.md)。

## 导入数据

导入部分允许您上传家谱文件或媒体档案。有关完整说明，请参见 [导入数据](import.md)。

## 媒体文件状态

此部分显示：

- 树中媒体对象的总数以及是否有任何对象缺少校验和
- 与服务器上缺失的文件关联的媒体对象数量

绿色勾号表示一切正常。如果检测到问题，将显示受影响对象的链接。缺失的校验和通常发生在从包含媒体引用但不包含实际文件的格式（如 GEDCOM）导入数据时。缺失的文件可以通过导入媒体档案功能上传。

## 导入媒体档案

允许上传媒体文件的 ZIP 文件以填补缺失的文件。有关完整说明，请参见 [导入数据](import.md)。

## 管理搜索索引

Gramps Web 维护一个全文搜索索引，通常在数据更改时自动更新。状态指示器显示当前索引的对象数量与总对象数量的关系。

单击 **更新搜索索引** 以触发完全重建。在后台运行任务时会显示进度指示器。通常只有在服务器升级后需要这样做。

### 语义搜索索引

如果服务器启用了 [语义（AI 驱动）搜索](../install_setup/configuration.md)，将出现一个额外部分，包含两个操作：

- **重新生成语义搜索索引** — 从头开始重建整个语义索引。这在计算上是昂贵的，可能需要很长时间。
- **更新语义搜索索引** — 执行增量更新，仅添加尚未索引的对象。比完全重建更快。

## 家谱名称

!!! note
    重命名树仅在 [多树设置](../install_setup/multi-tree.md) 中有效，或当 `TREE_ID` 在 [服务器配置](../install_setup/configuration.md) 中明确设置时。在默认的单树安装中，如果未设置 `TREE_ID`，将引发错误。

这允许更改底层 Gramps 家谱数据库的名称。输入新名称并单击 **重命名** 以应用。

## 检查和修复数据库

此工具检查 Gramps 数据库的内部不一致性，并修复可以修复的问题 — 类似于 Gramps Desktop 中的 [检查和修复数据库工具](https://www.gramps-project.org/wiki/index.php/Gramps_5.2_Wiki_Manual_-_Tools#Check_and_Repair_Database)。

单击 **检查和修复** 并等待进度指示器完成。结果显示在按钮下方：

- 如果未发现错误，将显示确认消息。
- 如果发现错误，将显示应用的修复摘要。

如果您遇到意外错误或可能由于数据库不一致性导致的行为（例如对象之间缺失的关系），请运行此工具。

## 危险区域

!!! danger
    危险区域中的操作是 **不可逆的**。继续之前请备份。

### 删除所有对象

从家谱中删除对象。单击 **删除** 会打开一个对话框，您可以选择删除：

- **所有对象** — 完全清除树
- **特定对象类型** — 例如，仅事件或仅媒体对象

您将被要求重新身份验证（再次登录）以确认该操作。删除作为后台任务运行，并显示进度指示器。

!!! warning
    仅删除某些对象类型（而不是一次性删除所有对象）可能会导致大型树花费很长时间，因为服务器必须检查和更新对象之间的所有关系。

!!! tip
    在导入新树之前，使用此功能重新开始，或删除导入不正确的特定对象类型。
