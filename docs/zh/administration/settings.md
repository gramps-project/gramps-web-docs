# 管理设置

**设置 > 管理** 页面可以通过应用程序顶部工具栏中的用户图标访问。该页面仅对具有所有者或管理员角色的用户可用，提供管理家谱数据库的工具。

该页面组织为可折叠的部分。单击部分标题以展开它。

## 数据

涵盖使用配额、导入数据和媒体文件管理。

### 使用配额

该部分顶部显示当前使用情况与任何配置限制的关系：

- **人员** – 家谱中的人员对象数量与配置的最大值（如果无限则为∞）
- **媒体存储** – 上传的媒体文件的总大小与配置的存储配额（如果无限则为∞）

配额由服务器管理员设置；有关详细信息，请参见 [服务器配置](../install_setup/configuration.md)。

### 导入数据

导入部分允许您上传家谱文件或媒体档案。有关完整说明，请参见 [导入数据](import.md)。

### 媒体文件状态

该部分显示：

- 家谱中媒体对象的总数以及是否有任何对象缺少校验和
- 与服务器缺失的文件相关的媒体对象数量

绿色勾号表示一切正常。如果检测到问题，将显示受影响对象的链接。缺失的校验和通常发生在从包含媒体引用但不包含实际文件的格式（如 GEDCOM）导入数据时。缺失的文件可以通过导入媒体档案功能上传。

### 导入媒体档案

允许上传媒体文件的 ZIP 文件以填补缺失的文件。有关完整说明，请参见 [导入数据](import.md)。

## 搜索索引

### 管理搜索索引

Gramps Web 维护一个全文搜索索引，该索引通常在数据更改时自动更新。状态指示器显示当前索引的对象数量与总对象数量的关系。

单击 **更新搜索索引** 以触发完全重建。在后台运行任务时会显示进度指示器。这通常仅在服务器升级后需要。

### 语义搜索索引

如果服务器启用了 [语义（AI 驱动）搜索](../install_setup/configuration.md)，将出现一个额外部分，包含两个操作：

- **重新生成语义搜索索引** – 从头开始重建整个语义索引。这是计算密集型的，可能需要很长时间。
- **更新语义搜索索引** – 执行增量更新，仅添加尚未索引的对象。比完全重建更快。

## 家谱设置

### 家谱名称

!!! note
    仅在 [多树设置](../install_setup/multi-tree.md) 中或在 [服务器配置](../install_setup/configuration.md) 中显式设置 `TREE_ID` 时，重命名树才有效。在默认的单树安装中，如果未设置 `TREE_ID`，将会引发错误。

这允许更改底层 Gramps 家谱数据库的名称。输入新名称并单击 **重命名** 以应用。

!!! tip
    如果您只想更改应用程序栏中显示的名称而不重命名数据库，请使用 [应用标题](#app-title) 设置。

### 研究者信息

设置主要研究者的姓名、地址和联系信息。这些信息嵌入在导出中（例如 GEDCOM 文件）。

## 自定义

### 主题颜色

为 Gramps Web 界面设置自定义 **主色** 和 **强调色**。这些颜色应用于该树的所有用户，并在保存后立即生效。

使用颜色选择器选择颜色，然后单击 **保存**。单击 **重置** 以恢复默认值。

### 应用标题

为应用程序设置自定义标题。如果设置，则会覆盖浏览器标题栏和顶部应用程序栏中的家谱名称。

输入标题并单击 **保存**。留空以使用默认值（家谱名称）。

### 首页备注

选择一个 Gramps **备注** 对象以在仪表板主页上显示。备注内容渲染在主仪表板列下方，所有有权访问该树的用户均可见。

使用对象选择器搜索并选择备注，然后保存。单击 **移除** 以清除当前主页备注。

### 首页图像

选择一个 Gramps **媒体** 对象以在仪表板主页上显示为图像。当与主页备注结合使用时，图像出现在备注文本旁边。没有备注时，仅显示图像。

使用对象选择器搜索并选择媒体对象，然后保存。单击 **移除** 以清除当前主页图像。

### 导出/导入设置

树级设置（应用标题、主题颜色、主页备注/图像等）可以导出为 JSON 文件以备份或复制到另一个 Gramps Web 实例。

- 单击 **导出设置** 以将当前设置下载为 JSON 文件。
- 单击 **导入树设置** 以上传先前导出的 JSON 文件并应用设置。

## 家谱处理

### 检查和修复数据库

该工具检查 Gramps 数据库的内部不一致性并修复可以修复的部分 – 类似于 Gramps Desktop 中的 [检查和修复数据库工具](https://www.gramps-project.org/wiki/index.php/Gramps_5.2_Wiki_Manual_-_Tools#Check_and_Repair_Database)。

单击 **检查和修复** 并等待进度指示器完成。结果显示在按钮下方：

- 如果未发现错误，将显示确认消息。
- 如果发现错误，将显示应用的修复摘要。

如果您遇到意外错误或可能由数据库不一致性引起的行为（例如对象之间缺失关系），请运行此工具。

## 标签

### 管理标签

为家谱创建、重命名、重新着色和删除 [标签](../user-guide/tags.md)。标签存储在 Gramps 数据库中，所有用户共享，并与 Gramps Desktop 完全兼容。

单击 **新标签** 创建标签。使用现有标签旁边的控件重命名（铅笔图标）、更改颜色（颜色选择器）或删除（删除图标）。

!!! note
    删除标签会将其从所有应用的对象中移除。

有关标签在 Gramps Web 中的使用情况，包括特殊的 `Blog` 和 `ToDo` 标签，请参见 [标签](../user-guide/tags.md)。

## 危险区

!!! danger
    危险区中的操作是 **不可逆的**。继续之前请备份。

### 删除所有对象

从家谱中移除对象。单击 **删除** 会打开一个对话框，您可以选择删除：

- **所有对象** – 完全清空树
- **特定对象类型** – 例如，仅事件或仅媒体对象

您将被要求重新验证（再次登录）以确认该操作。删除作为后台任务运行，并显示进度指示器。

!!! warning
    仅删除某些对象类型（而不是一次性删除所有对象）对于大型树可能需要很长时间，因为服务器必须检查和更新对象之间的所有关系。

!!! tip
    在导入新树之前使用此功能以重新开始，或删除导入不正确的特定对象类型。
